# 学生错题记录分析程序需求文档

## 1. 项目概述

### 1.1 项目目标

开发一款高效、易用的学生错题记录分析程序，核心目标包括：



* 帮助学生便捷记录错题、系统复习错题，精准定位知识薄弱点；

* 为教师提供班级学情数据支撑，实现针对性教学指导；

* 支持管理员对系统基础数据、用户权限进行统一管理，保障系统稳定运行。

### 1.2 核心用户



* 学生：错题录入、复习、查看个人错题分析；

* 教师：查看班级错题统计、学情分析、布置针对性练习；

* 管理员：用户管理、系统配置、数据维护。

### 1.3 运行环境



* 客户端：支持 PC 端（Chrome/Firefox/Edge 等主流浏览器）、移动端（iOS/Android 原生或 H5 适配）；

* 服务端：兼容 Java 生态主流技术栈（Spring Boot/Spring Cloud 等），支持 MySQL/Oracle 等关系型数据库。

## 2. 功能需求

### 2.1 学生模块

#### 2.1.1 错题录入



* 支持 3 种录入方式：

1. 手动录入：自定义输入题目内容（支持文本、公式编辑）、选项（单选 / 多选 / 填空 / 主观题适配）、正确答案、错误答案、错误原因；

2. 拍照识别：拍摄纸质错题，自动识别题目文本、选项、答案（支持公式、图片识别优化），支持手动修正识别结果；

3. Excel 导入：支持按模板批量导入错题（模板包含学科、知识点、题目类型、题目内容等必填字段）。

* 录入时需关联基础信息：学科（如数学 / 语文 / 英语）、年级、知识点（如数学 “二次函数”）、题目类型（单选 / 多选 / 填空 / 解答 / 实验）、难度等级（简单 / 中等 / 困难）、错误原因（如概念混淆 / 计算失误 / 审题不清 / 不会做）。

#### 2.1.2 错题管理



* 错题编辑：支持修改错题的题目内容、答案、关联信息等，支持补充错题解析、笔记；

* 错题分类：按学科、知识点、难度、错误原因、录入时间等维度自动分类，支持自定义标签分类（如 “月考错题”“期末重点”）；

* 错题删除：支持单条删除或批量删除，删除前需二次确认，支持回收站功能（误删后 7 天内可恢复）；

* 错题收藏：支持标记重点错题，单独生成 “收藏错题集”。

#### 2.1.3 错题复习



* 复习计划：基于艾宾浩斯遗忘曲线，自动生成个性化复习计划（首次复习、1 天后、3 天后、7 天后、15 天后），支持手动调整复习时间；

* 复习提醒：通过系统消息、短信（可选）提醒用户当日需复习的错题；

* 复习模式：

1. 自测模式：隐藏正确答案和解析，用户重新作答后比对答案，记录复习结果（正确 / 仍错误）；

2. 解析模式：直接展示题目解析、错误原因分析、知识点延伸；

* 复习记录：保存每次复习的时间、结果，支持查看复习历史轨迹。

#### 2.1.4 个人错题分析



* 数据统计：展示总错题数、已掌握错题数、未掌握错题数，按学科 / 知识点 / 错误原因 / 难度的错题分布占比（图表可视化：饼图、柱状图）；

* 薄弱点识别：自动分析高频错误知识点、高频错误原因，生成个人薄弱点报告；

* 错题导出：支持导出错题集为 Excel/PDF 格式（包含题目、答案、解析、错误原因等信息），可选择按学科 / 知识点 / 时间筛选导出内容。

### 2.2 教师模块

#### 2.2.1 班级管理



* 班级创建：支持创建多个班级，设置班级名称、年级、学科（班主任专属）；

* 学生管理：添加学生（支持批量导入 Excel，含学生姓名、学号、账号）、移除学生、查看班级学生列表；

* 权限设置：可指定班级副班主任，共享班级管理权限。

#### 2.2.2 班级错题统计



* 整体统计：查看班级总错题数、人均错题数，按学科 / 知识点 / 错误原因 / 难度的错题分布（图表可视化：折线图、热力图）；

* 高频错题排行：展示班级内出错次数最多的题目（按错题次数排序），查看每道题的错误学生名单、错误答案分布；

* 学生个体查看：点击学生姓名，查看该学生的错题详情、复习情况、个人薄弱点分析。

#### 2.2.3 学情分析



* 薄弱知识点定位：分析班级整体高频错误知识点，识别班级共同薄弱点；

* 学生分层：按错题掌握情况（已掌握 / 未掌握比例）将学生分为优秀 / 中等 / 待提升三层，支持查看分层学生名单；

* 趋势分析：展示班级错题数、薄弱知识点的变化趋势（按周 / 月统计），评估教学效果；

* 报告生成：自动生成班级学情分析报告（支持导出 PDF），包含统计数据、薄弱点、改进建议。

#### 2.2.4 针对性教学



* 练习布置：基于班级薄弱知识点，布置针对性练习题（可从系统题库选择或自定义添加），设置提交截止时间；

* 错题推送：可选择班级高频错题，推送给学生进行二次练习，跟踪练习完成情况；

* 批改反馈：查看学生提交的练习答案，支持批量批改 / 单题批改，添加文字反馈，标记优秀答案供学生参考；

* 答疑互动：接收学生提交的错题请教，在线回复解析或语音答疑，支持查看答疑历史。

### 2.3 管理员模块

#### 2.3.1 用户管理



* 用户列表：查看系统内所有学生、教师、管理员账号，展示账号状态（启用 / 禁用）、所属班级 / 角色；

* 账号操作：添加用户（手动创建或批量导入）、修改用户信息（姓名、密码重置）、禁用 / 启用账号、删除用户；

* 角色分配：为用户分配角色（学生 / 教师 / 管理员），支持批量分配角色权限。

#### 2.3.2 系统配置



* 基础数据维护：

1. 学科管理：添加 / 修改 / 删除学科（如语文、数学、物理），设置学科对应的年级；

2. 知识点管理：按学科层级维护知识点（如数学→代数→二次函数），支持批量导入 / 导出；

3. 字典配置：维护错误原因（概念混淆 / 计算失误等）、难度等级（简单 / 中等 / 困难）、题目类型（单选 / 多选等）的字典数据，支持自定义添加选项；

* 系统参数设置：设置复习提醒时间、错题回收站保留天数、Excel 导入模板下载地址、文件上传大小限制等。

#### 2.3.3 数据维护



* 数据备份：支持手动触发系统数据备份（含用户数据、错题数据、统计数据），设置自动备份周期（如每日凌晨）；

* 数据恢复：支持从备份文件恢复数据，恢复前需二次确认，记录恢复日志；

* 数据清理：清理过期数据（如超过 1 年的复习记录），支持按时间范围筛选清理内容。

#### 2.3.4 日志管理



* 操作日志：记录所有用户的关键操作（如错题录入、班级创建、权限修改），包含操作人、操作时间、操作内容、IP 地址；

* 登录日志：记录用户登录 / 登出信息，包含登录账号、登录时间、IP 地址、登录设备、登录状态（成功 / 失败）；

* 日志查询与导出：支持按时间、用户、操作类型筛选日志，导出为 Excel 格式。

## 3. 数据需求

| 实体名称                    | 核心字段                                                                                                                                                                                         | 字段说明                                                  |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| 用户表（user）               | user\_id、username、password、role、name、grade、class\_id、status、create\_time                                                                                                                     | role：0 - 学生 / 1 - 教师 / 2 - 管理员；status：0 - 禁用 / 1 - 启用 |
| 班级表（class）              | class\_id、class\_name、grade、subject、teacher\_id、create\_time                                                                                                                                 | teacher\_id：班主任 ID                                    |
| 学科表（subject）            | subject\_id、subject\_name、grade、status                                                                                                                                                       | status：0 - 未启用 / 1 - 启用                               |
| 知识点表（knowledge）         | knowledge\_id、knowledge\_name、subject\_id、parent\_id                                                                                                                                         | parent\_id：父知识点 ID（用于层级结构）                            |
| 错题表（error\_question）    | question\_id、user\_id、subject\_id、knowledge\_id、question\_type、difficulty、error\_reason、question\_content、options、correct\_answer、user\_answer、analysis、notes、tags、create\_time、update\_time | options：存储单选 / 多选选项（JSON 格式）                          |
| 复习计划表（review\_plan）     | plan\_id、question\_id、user\_id、plan\_time、actual\_time、review\_result                                                                                                                        | review\_result：0 - 未复习 / 1 - 复习正确 / 2 - 复习错误          |
| 练习表（exercise）           | exercise\_id、teacher\_id、class\_id、subject\_id、knowledge\_id、exercise\_name、deadline、create\_time                                                                                            | 教师布置的针对性练习                                            |
| 练习提交表（exercise\_submit） | submit\_id、exercise\_id、user\_id、answer\_content、submit\_time、score、feedback                                                                                                                 | feedback：教师批改反馈                                       |
| 系统日志表（system\_log）      | log\_id、user\_id、operation、operation\_content、ip\_address、operation\_time、status                                                                                                             | operation：操作类型（登录 / 错题录入等）                            |

### 3.2 数据约束



* 用户名唯一，密码采用 MD5 加密存储；

* 错题表中 question\_id 为主键，关联用户表 user\_id、学科表 subject\_id 等外键，支持级联查询；

* 所有核心数据（错题、用户信息）需支持数据完整性校验，避免空值或无效数据录入；

* 导出文件（Excel/PDF）需保持数据格式统一，支持中文乱码处理。

## 4. 非功能需求

### 4.1 性能需求



* 响应速度：页面加载时间≤2 秒，错题查询、统计分析结果返回时间≤1 秒，拍照识别题目时间≤3 秒；

* 并发支持：支持至少 500 名用户同时在线操作，单班级 100 名学生同时提交练习无卡顿；

* 数据存储：支持单用户错题数≥1000 条，单班级错题数≥10000 条，系统总数据量可扩展至百万级。

### 4.2 安全性需求



* 权限控制：不同角色仅能访问对应功能模块，学生无法查看其他学生错题，教师仅能查看所属班级数据；

* 数据安全：用户密码加密存储，敏感数据（如学生信息）传输过程中采用 HTTPS 加密；

* 防恶意操作：限制同一账号高频登录（10 分钟内失败≥5 次锁定 30 分钟），防止 SQL 注入、XSS 攻击等常见安全漏洞；

* 数据备份：自动备份周期≤24 小时，备份数据保留≥30 天，支持异常情况下的数据恢复。

### 4.3 易用性需求



* 界面设计：简洁明了，导航清晰，核心功能（错题录入、复习）操作步骤≤3 步；

* 交互体验：支持快捷键操作（如错题录入时 Ctrl+S 保存），表单提交时实时校验数据，给出明确错误提示；

* 学习成本：无需专业培训，学生、教师可快速上手，提供操作指南（帮助文档 / 视频教程）；

* 响应式适配：PC 端、移动端界面自动适配，移动端支持手势操作（如滑动切换复习题目）。

### 4.4 可扩展性需求



* 技术扩展：支持后续集成第三方题库、在线支付（增值服务）、家校通通知等功能；

* 功能扩展：预留 API 接口，支持对接学校教务系统（如学生信息同步）、学习终端（如学习机）；

* 定制化支持：允许学校 / 班级自定义错题分类标签、学情报告模板等。

## 5. 界面需求

### 5.1 整体设计原则



* 风格统一：采用简洁、清新的设计风格，色彩以蓝色（信任）、白色（简洁）为主，避免过多装饰；

* 导航清晰：顶部导航栏展示核心模块（错题管理、复习计划、分析报告等），侧边栏支持快速切换功能；

* 响应式设计：适配不同屏幕尺寸（PC 端≥1280px，平板≥768px，手机≥375px），避免内容溢出。

### 5.2 核心页面要求



* 登录页：支持账号密码登录、验证码登录，提供 “忘记密码” 找回功能；

* 学生首页：展示今日复习提醒、个人错题统计概览、薄弱点推荐，快速入口（错题录入、复习计划）；

* 错题录入页：分步骤引导录入，支持切换录入方式（手动 / 拍照 / 导入），实时保存草稿；

* 教师学情分析页：以图表为主、文字为辅展示统计数据，支持筛选时间范围（周 / 月 / 学期）、下载报告；

* 管理员后台：采用列表 + 表单的布局，支持批量操作、快速搜索，数据展示清晰直观。

## 6. 其他需求

### 6.1 导出导入功能



* 支持 Excel 格式的错题批量导入 / 导出，提供标准模板下载；

* 支持 PDF 格式的学情报告、错题集导出，保持排版整齐、可打印；

* 导入数据时支持格式校验，提示错误数据行及原因，允许修正后重新导入。

### 6.2 提醒功能



* 复习提醒：系统消息（站内信）、移动端推送，可选短信提醒（需用户开通）；

* 练习提醒：教师布置练习后，提醒学生及时完成；练习截止前 1 天，提醒未提交学生；

* 系统通知：管理员发布的系统更新、功能调整等通知，支持已读 / 未读标记。

### 6.3 搜索功能



* 支持多维度搜索错题（关键词、学科、知识点、错误原因、录入时间）；

* 支持模糊搜索（如输入 “二次” 匹配 “二次函数” 相关错题）；

* 搜索结果支持排序（按录入时间、错误次数）、筛选（按难度、题目类型）。
